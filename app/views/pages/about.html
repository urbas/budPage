Bud

The build tool without the "ill".

You can try Bud out now.

Install Bud via Chocolatey!

Automates build-related processes

From downloading dependencies, compiling your code, testing, deployment, releases. You name it. Now you can do it with Bud. Just write some C#.

Super fast, highly customisable, and modern

Bud's build configuration is plain C# code. It's therefore as fast as normal C# programs and as customisable. Bud uses great concepts used in other modern build tools (for example: task pipelines, convention over configuration, dependency and package management, etc.).

Easy to setup

For simplest projects you don't even need anything. Just put your C# code into src/main/cs and run bud. This will compile your code and generate an EXE file.

Define your builds in C#

You can configure your build in the file ./bud/Build.cs.

Here's an example of a Build.cs file, where we define a simple EXE project with a NuGet dependency: 


using Bud;
using Bud.CSharp;
using Bud.Projects;

public class Build : IBuild {
  public Settings Setup(Settings settings, string baseDir) {
    return settings.Project("Foo", baseDir, Cs.Exe(
        Cs.Dependency("Newtonsoft.Json")
    ));
  }
}

Now place your C# sources into the src/main/cs folder and start coding.

If you now invoke bud run in the root directory of your project, Bud will compile your code and run the executable.
